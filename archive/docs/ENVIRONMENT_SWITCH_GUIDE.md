# 环境切换功能使用指南

## 功能概述

现在Binance交易机器人支持在测试环境和生产环境之间切换。你可以在Telegram Bot面板中直接切换环境，无需重启机器人。

## 环境配置

### 1. 环境变量设置

更新你的 `.env` 文件，添加以下配置：

```bash
# ========================
# Binance API 配置
# ========================

# 生产环境API密钥（必须设置）
BINANCE_API_KEY=你的主网API密钥
BINANCE_API_SECRET=你的主网API密钥

# 测试环境专用API密钥（可选，如果设置则使用专用测试密钥）
BINANCE_TESTNET_API_KEY=你的测试网API密钥
BINANCE_TESTNET_API_SECRET=你的测试网API密钥

# 默认使用测试环境（true: 测试环境, false: 生产环境）
# 注意：机器人启动时会使用这个设置，但可以在面板中切换
BINANCE_TESTNET=true

# ========================
# Telegram Bot 配置
# ========================

TELEGRAM_BOT_TOKEN=你的Telegram Bot Token
TELEGRAM_ALLOWED_USER_IDS=你的用户ID

# ========================
# 交易默认参数
# ========================

DEFAULT_SIDE=long
DEFAULT_LEVERAGE=5
DEFAULT_MARGIN_USDT=50
DEFAULT_TP_PCT=5
DEFAULT_SL_PCT=2
```

### 2. 配置说明

- **BINANCE_API_KEY / BINANCE_API_SECRET**: 主网（生产环境）API密钥，必须设置
- **BINANCE_TESTNET_API_KEY / BINANCE_TESTNET_API_SECRET**: 测试网专用API密钥（可选）
  - 如果设置：测试环境使用专用测试密钥
  - 如果未设置：测试环境使用主网API密钥（不推荐）
- **BINANCE_TESTNET**: 机器人启动时的默认环境

## 使用方法

### 1. 启动机器人

```bash
./run.sh
```

### 2. 在Telegram中打开面板

发送 `/start` 或 `/panel` 命令，你会看到新的面板界面：

```
📊 交易配置  🧪 测试环境
━━━━━━━━━━━━━━━
🔐 开仓锁: 🔒 已锁定
🧭 方向: `↗️ LONG`
⚡ 杠杆: `5x`
💰 保证金: `50 USDT`
🎯 止盈: `+5%`
🛡 止损: `-2%`

需先解锁才能点币种开仓
环境: testnet (测试环境专用密钥)
```

### 3. 环境切换按钮

在面板中，你会看到一个新的按钮：

- **测试环境**: 显示当前环境状态
- **点击该按钮**: 进入环境设置菜单

### 4. 环境设置菜单

点击环境按钮后，进入环境设置菜单：

```
🌍 环境设置

🧪 测试环境 ✓
💰 生产环境

── 环境信息 ──

测试环境: ✅ 专用测试密钥
生产环境: ✅ 主网密钥已设置

⬅ 返回配置
```

### 5. 切换环境

1. **切换到生产环境**:
   - 点击"💰 生产环境"按钮
   - 系统会检查是否设置了主网API密钥
   - 如果已设置，显示确认对话框
   - 确认后切换到生产环境

2. **切换到测试环境**:
   - 点击"🧪 测试环境"按钮
   - 立即切换到测试环境

### 6. 生产环境切换确认

当切换到生产环境时，会显示确认对话框：

```
⚠️ 确认切换到生产环境

生产环境使用真实资金交易！
请确认：
1. 已设置正确的生产环境API密钥
2. 了解生产环境交易风险
3. 确认当前没有测试环境的未平仓位

[✅ 确认切换到生产环境] [❌ 取消]
```

## 安全特性

### 1. 环境切换验证

- **切换到生产环境**: 必须已设置 `BINANCE_API_KEY` 和 `BINANCE_API_SECRET`
- **环境密钥隔离**: 测试环境优先使用专用测试密钥
- **安全警告**: 生产环境切换需要二次确认

### 2. 环境信息显示

面板底部显示当前环境信息：

- 环境类型: `testnet` 或 `mainnet`
- API密钥来源: `测试环境专用密钥` 或 `主网密钥`

### 3. 交易安全

- **开仓锁**: 两个环境共享开仓锁设置
- **参数独立**: 两个环境的交易参数（杠杆、保证金等）独立保存
- **持仓隔离**: 测试环境和生产环境的持仓完全隔离

## 最佳实践

### 1. 测试环境配置

```bash
# 获取测试环境API密钥
1. 访问 https://testnet.binancefuture.com
2. 注册测试账号
3. 生成API密钥
4. 在.env文件中设置：
   BINANCE_TESTNET_API_KEY=你的测试密钥
   BINANCE_TESTNET_API_SECRET=你的测试密钥
```

### 2. 生产环境准备

```bash
# 切换到生产环境前
1. 确保已设置正确的主网API密钥
2. 确认API密钥有合约交易权限
3. 设置IP白名单（推荐）
4. 小额测试：先用最小保证金测试
```

### 3. 环境切换流程

```
测试环境 → 生产环境
1. 在测试环境完成所有测试
2. 平掉所有测试仓位
3. 切换到生产环境
4. 小额测试生产环境功能
5. 正常交易

生产环境 → 测试环境
1. 平掉或管理好生产环境仓位
2. 切换到测试环境
3. 测试新功能或策略
```

## 故障排除

### 1. 无法切换到生产环境

**问题**: 点击"生产环境"按钮无反应或报错

**解决**:

```bash
# 检查主网API密钥
echo $BINANCE_API_KEY
echo $BINANCE_API_SECRET

# 如果未设置，在.env文件中添加
BINANCE_API_KEY=你的主网密钥
BINANCE_API_SECRET=你的主网密钥

# 重启机器人
./run.sh
```

### 2. 环境切换后交易失败

**问题**: 切换环境后开仓失败

**解决**:

1. 检查当前环境的API密钥是否正确
2. 检查网络连接
3. 检查API密钥权限
4. 查看日志文件 `bot.log`

### 3. 环境显示不正确

**问题**: 面板显示的环境与实际不符

**解决**:

1. 发送 `/refresh` 命令刷新面板
2. 重启机器人
3. 检查 `settings.json` 文件中的环境设置
